### 架构特点

- **微服务架构**: 前后端分离，服务解耦
- **RESTful API**: 标准化接口设计
- **响应式设计**: 支持多设备访问
- **模块化开发**: 组件化、可复用
- **数据驱动**: 基于数据的智能决策

## 🛠️ 技术选型

### 前端技术栈

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| **Vue 3** | 3.x | 组合式API，更好的TypeScript支持，性能优化 |
| **Vite** | 4.x | 快速构建，热更新，ES模块支持 |

| **Pinia** | 2.x | 新一代状态管理，更好的TypeScript支持 |
| **Axios** | 1.x | 成熟的HTTP客户端，拦截器支持 |
| **Element Plus** | 2.x | 丰富的组件库，Vue3原生支持 |

## 🔄 数据流设计

### 路径规划数据流

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant J as Java后端
    participant A as 高德地图API
    participant D as 数据库
    
    U->>F: 输入起点终点
    F->>J: POST /api/route/plan
    J->>A: 调用路径规划API
    A->>J: 返回路径数据
    J->>D: 保存路径记录
    J->>F: 返回路径结果
    F->>U: 显示路径地图
```

### 装备推荐数据流

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端
    participant P as Python后端
    participant E as 电商平台
    participant D as 数据库
    
    U->>F: 搜索装备
    F->>P: GET /api/equipment/search
    P->>D: 查询本地数据
    alt 数据不存在或过期
        P->>E: 爬取最新数据
        E->>P: 返回装备信息
        P->>D: 更新数据库
    end
    P->>F: 返回装备列表
    F->>U: 显示推荐结果
```

## 🔒 安全架构

### 认证授权机制

```
┌─────────────────────────────────────┐
│              前端安全               │
│  ┌─────────────────────────────────┐ │
│  │  - Token存储 (localStorage)      │ │
│  │  - 路由守卫 (Route Guards)      │ │
│  │  │  请求拦截 (Axios Interceptor) │ │
│  │  - XSS防护 (Content Security)   │ │
│  └─────────────────────────────────┘ │
├─────────────────────────────────────┤
│              后端安全               │
│  ┌─────────────────────────────────┐ │
│  │  - JWT Token验证                │ │
│  │  - CORS跨域配置                 │ │
│  │  - 参数验证 (Validation)        │ │
│  │  - SQL注入防护                  │ │
│  │  - 接口限流 (Rate Limiting)     │ │
│  └─────────────────────────────────┘ │
├─────────────────────────────────────┤
│              数据安全               │
│  ┌─────────────────────────────────┐ │
│  │  - 数据库连接加密                │ │
│  │  - 敏感数据脱敏                  │ │
│  │  - 定期备份策略                  │ │
│  │  - 访问权限控制                  │ │
│  └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

### 安全策略

1. **身份认证**
   - JWT Token机制
   - Token自动刷新
   - 多设备登录管理

2. **权限控制**
   - 基于角色的访问控制(RBAC)
   - 接口权限验证
   - 数据权限隔离

3. **数据保护**
   - HTTPS传输加密
   - 敏感数据加密存储
   - 个人信息脱敏
