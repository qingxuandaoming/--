## 📋 开发流程概览

```
需求分析 → 技术设计 → 创建功能分支 → 编写代码 → 单元测试 → 集成测试 → 代码审查 → 合并到 develop → 部署测试环境 → 功能测试 → 发布
```

### 分支命名规范

| 分支类型 | 命名格式 | 示例 |
|----------|----------|------|
| 功能开发 | `feature/功能描述` | `feature/user-authentication` |
| Bug 修复 | `bugfix/问题描述` | `bugfix/login-error` |
| 热修复 | `hotfix/问题描述` | `hotfix/security-patch` |
| 发布 | `release/版本号` | `release/v1.2.0` |

---

## 🔐 安全开发提示

- 所有用户输入必须经过验证和转义
- 敏感数据（密码、Token）禁止硬编码
- API 接口必须做权限校验
- 数据库操作使用参数化查询
- 生产环境禁用 Swagger 等调试工具

详细安全规范请参阅 [03-安全指南](./03-安全指南.md)。
